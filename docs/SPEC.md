# SPEC

## 一句話描述

個人用的漫畫追蹤工具,記錄每部漫畫看到第幾話 / 第幾卷,以及看完之後的推薦指數。

## 使用者

- 只有我自己一人使用
- 多裝置(電腦、手機)會用,但階段 1 還沒接後端,純單機
- 不需要登入系統(階段 4 才會加上單一密碼保護)

## 核心功能

- 漫畫列表(卡片式)
- 新增 / 編輯 / 刪除漫畫
- 更新進度:**卷與話獨立記錄**(同一部漫畫可以同時有「第 12 卷」「第 98 話」)
- 切換 status(想看 / 追讀中 / 棄追 / 已追完)
- 已追完才能評分(1–5 星)
- 改 status 與填評分是**兩個獨立動作**,不綁在一起
- **分類**(階段 1.5 加入):每部漫畫一個分類,5 選 1(熱血 / 懸疑 / 冒險 / 愛情 / 其他)
- 首頁可同時依「狀態」與「分類」兩列獨立 chip 篩選

## 明確不做

- 社交、分享、留言
- 推薦演算法、AI 推薦
- 自動爬蟲抓更新
- 多人協作 / 帳號管理
- 自訂分類、多標籤(目前 5 個固定 enum,如未來要改成自訂或多選需另立資料模型)
- 搜尋(階段 1 不做,之後可能加)
- localStorage / IndexedDB 持久化(階段 1 故意不做)

## 階段 1 / 1.5 範圍

- 純前端 Vue 3 + TypeScript prototype
- 資料用記憶體中的 mock 陣列(reload 後資料消失,這是預期行為)
- 不接任何後端 API
- 不部署,本機 `npm run dev` 跑就好
- 階段 1.5:加分類欄位 + 分類 badge + 分類篩選 chips
