# UI Spec

## 頁面清單(階段 1 / 1.5)

- 主頁(列表)
- 新增漫畫 modal
- 編輯互動採**就地編輯**(in-place):點數字編輯、點星評分,不開另一個頁面 / 大 modal

## 主頁(列表)

### 頂部

- 左側:標題「我的漫畫」+ 統計列(共 X 部 · 追讀中 Y · 已追完 Z · 棄追 W)
- 右側:「＋ 新增漫畫」按鈕

### 篩選 chips(階段 1.5 起共兩列)

- **狀態列**:全部 / 想看 / 追讀中 / 棄追 / 已追完(預設「全部」)
- **分類列**:全部 / 熱血 / 懸疑 / 冒險 / 愛情 / 其他(預設「全部」)
- 兩列**獨立**篩選,效果為 AND(同時生效)
- 每列前面有小標籤(「狀態」「分類」)幫助識別

### 卡片網格

- CSS Grid: `repeat(auto-fit, minmax(200px, 1fr))`,gap 12px
- 卡片:白底、`border-radius-lg`、0.5px 邊框、padding 1rem 1.25rem

## 卡片內容(依 status 不同呈現)

### 想看 / 追讀中 / 棄追

```
┌─────────────────────────────────┐
│ 書名      [分類][狀態][⋯]      │
│                                 │
│ 卷  [12]                        │
│ 話  [98]                        │
│                                 │
│ ──────────────────────────────│
│ 3 天前                          │
└─────────────────────────────────┘
```

- 標題列右側:**分類 badge → 狀態 badge → 「⋯」選單**(由左到右)
- 顯示卷 / 話兩行(獨立)
- 數字底色 secondary,可點擊編輯
- 沒填的欄位顯示「—」灰色佔位,可點擊開始填
- 棄追的卡片整體 `opacity: 0.75`(視覺退一階)

### 已追完

```
┌─────────────────────────────────┐
│ 書名      [分類][狀態][⋯]      │
│                                 │
│ 推薦指數                        │
│ ★★★★☆                          │
│                                 │
│ ──────────────────────────────│
│ 完讀時間                        │
└─────────────────────────────────┘
```

- **不顯示**卷 / 話
- 中央顯示「推薦指數」標籤 + 五顆星
- 沒評分時顯示「點此評分」(虛線底線、灰字、可點擊)
- 點空星填、點已填的星可改

### 狀態 badge 顏色對照

| Status | 對應色階(Tailwind) |
|---|---|
| 想看 | `bg-neutral-100 text-neutral-600` |
| 追讀中 | `bg-blue-50 text-blue-700` |
| 棄追 | `bg-red-50 text-red-700` |
| 已追完 | `bg-green-50 text-green-700` |

### 分類 badge 顏色對照(階段 1.5)

刻意比狀態 badge 柔和一階,讓狀態仍是主視覺、分類是輔助標示。

| Category | 對應色階(Tailwind) |
|---|---|
| 熱血 | `bg-orange-50 text-orange-700` |
| 懸疑 | `bg-violet-50 text-violet-700` |
| 冒險 | `bg-emerald-50 text-emerald-700` |
| 愛情 | `bg-pink-50 text-pink-700` |
| 其他 | `bg-neutral-100 text-neutral-600` |

## 互動細節

### 編輯卷 / 話數字

- 顯示狀態:數字底色 secondary
- **點擊** → 變 input(數字鍵盤,寬 64px,高 28px)
- **Enter 鍵或失焦(blur)** → 儲存
- **Esc 鍵** → 取消(回到原值)
- 樂觀更新:UI 立刻變,失敗回滾並提示
- 任何更新會**同時更新 `lastReadAt` 和 `updatedAt`**

### 填 / 改評分(僅 completed)

- 已有評分:顯示星星(實心 + 空心),點任一顆可改
- 沒有評分:顯示「點此評分」灰色虛線連結
- **點擊「點此評分」** → 就地展開 5 顆空星,點選一顆即儲存
- 點同一顆星可清除評分(回到「點此評分」)

### 切換 status

- 卡片 badge 不直接點擊改 status(避免誤觸)
- 切換方式:卡片右上「⋯」選單
- 切換 status:打開一個小 dropdown 選 4 種之一
- 改 status **不影響**其他欄位(進度、評分、分類原封不動保留)

### 編輯分類(階段 1.5)

- **目前**:只能在「新增漫畫 modal」設分類,卡片上**不可就地編輯**
- 卡片上的分類 badge 純顯示用,不互動
- 想改分類 → 刪除重新加(暫時權宜),或等未來補上「分類就地編輯」
- 若使用一段時間發現需要改分類很頻繁,再加進 BACKLOG

### 新增漫畫(modal)

- 點頂部「＋ 新增漫畫」按鈕開啟
- 欄位:
  - 書名(必填)
  - status(下拉,預設「想看」)
  - **分類**(下拉,預設「其他」)
  - 卷(可空,只在 status 不是 completed 時顯示)
  - 話(可空,只在 status 不是 completed 時顯示)
  - 評分(可空,只在 status 是 completed 時顯示)
- 「取消」與「新增」按鈕
- 新增後關閉 modal,新卡片出現在列表

### 刪除

- 從卡片「⋯」選單觸發
- 跳一個確認對話框「確定刪除《書名》?」,有「取消」/「刪除」

## 排序

- 預設按 `lastReadAt` desc(最近動過進度的排前面)
- 階段 1 不做手動排序,等之後

## 響應式

- 桌面:卡片網格 `minmax(200px, 1fr)` 自動排
- 手機:同樣的 grid,因為 minmax 會自然降到單欄(寬度不夠時)
- 沒有單獨的手機版 layout
